Building a Story and Game Using Scratch 3

Scratch 3 is a beginner-friendly platform for creating interactive stories and games using block-based coding. It allows users to design characters, set up backgrounds, and add animations with simple drag-and-drop commands. In our project, we had developed three different Scratch programs: a Maze Game, an Apple Catching Game, and a Fun Oreo Story.

These projects helped us understand event handling, loops, conditionals, and sprite interactions, making Scratch 3 a great tool for creative learning!

Maze Game

The Maze Game challenges players to navigate through a maze while avoiding obstacles. Using arrow keys, the player moves a sprite to reach the goal. The game includes boundaries and logic to detect collisions.

Game Features:

  • Interactive maze navigation using arrow keys
  • Collision detection with maze walls
  • Goal-reaching mechanics
  • Game restart functionality
  • Score tracking system

Key Code Blocks:

  • Movement controls using "when key pressed" blocks
  • "If touching color" conditions for wall detection
  • Position reset on collision
  • Victory condition when reaching the goal

Apple Catching Game

The Apple Catching Game is a fun challenge where a basket moves left and right to catch falling apples. We used loops and conditional statements to increase difficulty as the game progresses.

Game Features:

  • Basket control using left and right arrow keys
  • Randomly falling apples at varying speeds
  • Score increment when catching apples
  • Progressive difficulty increase
  • Game over condition when missing apples

Key Code Blocks:

  • "Forever" loops for continuous apple generation
  • Random position selection for apple spawning
  • "If touching" conditions for catch detection
  • Variable manipulation for score tracking
  • Conditional statements for difficulty adjustment

Fun Oreo Story

The Fun Oreo Story is an interactive storytelling project where characters engage in a playful Oreo adventure. It uses dialogues, sound effects, and animations to make the story engaging.

Story Features:

  • Multiple animated characters
  • Interactive dialogue sequences
  • Scene transitions with background changes
  • Sound effects and background music
  • User interaction points throughout the story

Key Code Blocks:

  • "Say" blocks for character dialogue
  • "Wait" blocks for timing control
  • "Switch backdrop" for scene changes
  • "Play sound" for audio effects
  • "Broadcast" messages for synchronizing events

Building an Attendance App Using MIT App Inventor

MIT App Inventor is a user-friendly platform for developing mobile applications using block-based coding. I used this tool to create an Attendance App with a face camera feature to capture attendance along with the date and time. The app allows users to click a picture for verification and stores the data automatically.

App Features:

  • Camera integration for taking photos
  • Automatic date and time stamping
  • Database storage for attendance records
  • User-friendly interface with simple controls
  • Image display of captured attendance

Key Components Used:

  • Camera component for photo capture
  • Clock component for timestamp generation
  • TinyDB for local data storage
  • Button components for user interaction
  • Image components for displaying captured photos
